BadGuyMaker

Build an enemy definition and save it as JSON. Drop the saved file into enemies/<name>/ at the repo root, alongside the sprite sheet image(s) it references. Uploaded sheets get their background color knocked out and are auto-sliced with the same slicer the game uses, so what you see here is what the game will see.

Every sliced row of the enemy sheet plays below. Name the rows you want to keep (blank = not exported) and set an fps for each. Named rows are saved into the JSON as animations, using the same [row, index] convention as the player sprite maps.
